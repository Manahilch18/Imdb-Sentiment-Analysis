An end-to-end Machine Learning pipeline developed to classify binary sentiment (Positive/Negative) on the scale of the **IMDB 50K Movie Reviews Dataset**. This repository documents the comparative bench-marking of three foundational Naive Bayes architectural variants (`MultinomialNB`, `BernoulliNB`, and `GaussianNB`), highlighting feature engineering trade-offs and structural mathematical constraints in text classification.

---

## 🛠️ 1. Project Architecture & Pipeline Design

The system implements a production-focused text-cleansing blueprint before projecting textual reviews into high-dimensional numerical space.
[Raw Web Text] ──> [HTML Strip & Regex Lowering] ──> [Custom Negation Preservation]
│
[Model Evaluation] <── [TF-IDF Feature Space] <── [Porter Stemming Target]

### Key Preprocessing Strategies:
1. **Noise Abatement:** Strips persistent web-scraping markup (`<br />`) via `BeautifulSoup` and purges symbols using optimized alphanumeric regular expressions (`regex`).
2. **Custom Semantic Anchoring:** Standard stop-word engines blindly eliminate polarity-switching tokens. This implementation overrides NLTK defaults to protect negation anchors (`not`, `no`, `never`, `didnt`), preventing absolute context inversion.
3. **Morphological Compacting:** Implements `PorterStemmer` to compress lexical variants down to statistical root tokens, shrinking feature space dimensionality efficiently.

## 🧠 3. Critical Theoretical Insights (The Architecture Breakdown)

A core focus of this project is demonstrating a clear understanding of math-to-feature alignments for recruiters and technical contributors:

### ⚠️ The TF-IDF vs. GaussianNB Architectural Mismatch
While `MultinomialNB` and `BernoulliNB` achieved excellent performance bounds, `GaussianNB` lagged significantly behind. This is a direct consequence of a data format mismatch:
* **The Sparse Matrix Realism:** Text representation via TF-IDF outputs massive, memory-efficient **Sparse Matrices** containing a heavy density of zeros. 
* **The Continuous Assumption Constraint:** `GaussianNB` inherently assumes that features are continuous and follow a standard Gaussian (Normal) distribution. To map sparse data into it, matrices must be forcefully converted to dense format arrays via `.toarray()`. This step violates text sparsity, introduces extreme memory bloat (risking production RAM crashes), and forces an artificial distribution model onto discrete word counts, leading to inherently lower accuracy.

### 📈 Optimization Mechanisms:
* **Laplace Smoothing ($\alpha = 0.5$):** Tuned smoothing parameters prevent the zero-probability problem on out-of-vocabulary (OOV) terms present in unseen testing partitions, maximizing generalization capabilities.
* **Feature Interpretability:** Extracting mathematical weights using `tfidf.get_feature_names_out()` verified that the model systematically relies on isolated negation components (`not`, `no`) to identify negative arrays, confirming data cleaning validity.
